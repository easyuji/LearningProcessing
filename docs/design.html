リスクと対策

高
GitHubトークンがgit logに残る
push時にトークン入りURLが remote tracking に記録される場合がある
push後に git remote set-url origin https://github.com/... でURLをリセット（現状未実装）
高
Edge TTSが突然使えなくなる
Microsoftの仕様変更で403エラーが発生。過去に実際に発生済み
edge-ttsのバージョンを最新に保つ（requirements.txt: edge-tts>=7.2.8）。OpenAI TTSに切り替えも可能
中
docs/audioが大きくなりすぎる
GitHubリポジトリは1ファイル100MB、リポジトリ全体1GBが推奨上限。MP3は1本20〜37MB
cleanup.pyが90日超のファイルを自動削除。現在6本≈150MBで問題なし
中
Macがスリープ中は実行されない
launchdはMac起動中のみ動作。スリープ時は7時にならない
手動で python3 main.py を実行するか、起動後に自動実行（RunAtLoad=true）に変更可能
中
GmailアプリパスワードはGoogleアカウント全体にアクセス可能
アプリパスワードはIMAPだけでなく他のGoogleサービスにも使える強い権限
キーチェーンに保存、.envや会話中には流さない（実施済み）。不審な場合はGoogleアカウントで無効化可能
低
RSSフィードが壊れる
バグで既存エピソードが消えることがあった（今回修正済み）
recover_feed.pyで全エピソードを再構築できる（実際に使用）
低
GitHub Pagesの反映遅延
git push後、実際にURLで聴けるまで数分かかることがある
待てばOK。通常2〜5分以内に反映
